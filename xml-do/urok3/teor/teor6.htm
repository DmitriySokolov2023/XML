Урок 3. Проверка документов XML с использованием DTD
 

6. Определение обязательности и количества элементов

 

При задании контентной модели можно указать «обязательность» указания дочерних элементов. Это требуется для случая, при котором некоторые дочерние элементы не являются обязательными для отдельных родительских элементов, а некоторые – могут повторяться. Например, элемент DESCRIPTION не является обязательным для всех элементов TOVAR. Также очень важно, что некоторые дочерние элементы могут повторяться, например, в родительском элементе TOVARS может быть несколько дочерних элементов TOVAR, а в родительском элементе TOVAR дочерние элементы NAIM и PRICE может быть только единственным.

Для этого после указания имени дочернего элемента требуется указать один из символов, задающих обязательность и «повторяемость» элемента.

Символы перечислены в таблице

Символ (название)

Описание

? ( вопросительный знак)

Элемент необязателен, если задан, то только однократно

* (звездочка)

Элемент необязателен, может повторяться многократно

+ (плюс)

Элемент обязателен, может повторяться многократно

Символ отсутствует(по умолчанию)

Элемент обязателен, может быть только однократно

 

Эти символы могут применяться как к отдельному элементу, так и к группе элементов, заданной в скобках.

Примеры использования:

Требуется задать прайс-лист из нескольких товаров. XML документ будет следующего вида:

<?xml version="1.0" encoding="windows-1251" ?>

<TOVARS>

<TOVAR>

<NAIM>Бензин</NAIM>

<PRICE>20</PRICE>

</TOVAR>

<TOVAR>

<NAIM> Керосин </NAIM>

<PRICE>40</PRICE>

</TOVAR>

</TOVARS>

Схема DTD, соответствующая этому документу будет иметь следующий вид:

<!DOCTYPE TOVARS [

<!ELEMENT TOVARS (TOVAR*)>

<!ELEMENT TOVAR (NAIM,PRICE)>

<!ELEMENT NAIM (#PCDATA)>

<!ELEMENT PRICE (#PCDATA)>

]>

Значение #PCDATA свидетельствует, что указанный элемент не является составным (содержит только текстовую информацию).
