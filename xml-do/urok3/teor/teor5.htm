Урок 3. Проверка документов XML с использованием DTD
 

5. Порядок объявления элемента

 

Элементы являются основой для построения схемы DTD. Для объявления элемента используется следующее предложение:

<!ELEMENT element_name content_model>

element_name - имя элемента

content _ model – указание содержимого элемента (контентной модели). Возможны 4 вида контентной модели:

Пустой элемент. Этот элемент не содержит дочерних элементов или информационного содержания. Такой элемент может содержать только атрибуты.

Пример такого элемента: <TOVAR NAIM=”Бензин” PRICE =”20” />

Для определения пустого элемента в схеме DTD используется такое предложение:

<!ELEMENT TOVAR EMPTY>

Элемент, содержащий дочерние элементы. В этом случае возможны варианты: может содержаться только один элемент, несколько элементов с конкретной структурой или несколько элементов с возможностью выбора. В любом случае имена дочерних элементов помещаются в круглые скобки () .Если элемент содержит несколько дочерних элементов, их имена помещаются в скобки и разделяются запятыми. Если возможен выбор подчиненных элементов, используется вертикальный знак |

Примеры:

Элемент TOVAR содержит один дочерний элемент с именем NAIM . В схеме он будет описан следующим образом:

<!ELEMENT TOVAR (NAIM)>

Элемент TOVAR содержит два дочерних элемента с именами NAIM и PRICE. В схеме этот элемент будет описан следующим образом:

<!ELEMENT TOVAR (NAIM, PRICE )>

Дочерние элементы должны быть расположены строго в заданной последовательности, то есть так:

<TOVAR>

<NAIM>Бензин</NAIM>

<PRICE>20</PRICE>

</TOVAR>

При расположения элементов в другой последовательности, например:

<TOVAR>

<PRICE>20</PRICE>

<NAIM>Бензин</NAIM>

</TOVAR>

При проверке соответствия структуры документа заданной схеме (например, в случае открытия документа в браузере) появится сообщение об ошибке.

Иногда возникают ситуации, при которых возможен выбор при задании структуры документа. Например, в прайс-листе для некоторых товаров может быть определен элемент DESCRIPTION, в котором будет помещено текстовое описание данного товара.

В этом случае допустимыми являются обе структуры:

<TOVAR>

<NAIM>Бензин</NAIM>

<PRICE>20</PRICE>

</TOVAR>

и

<TOVAR>

<NAIM> Керосин </NAIM>

<PRICE>20</PRICE>

<DESCRIPTION> Авиационный </DESCRIPTION>

</TOVAR>

В этом случае в схеме потребуется указать возможность выбора дочерних элементов. Для этого используется такое описание:

<!ELEMENT TOVAR ((NAIM,PRICE) | (NAIM,PRICE,DESCRIPTION))>

Обратите внимание на двойные скобки – их использование в этом случае обязательно.

В некоторых случаях невозможно определиться с дочерними элементами – они могут быть различными и не подчиняться никаким правилам. В этом случае для описания используется слово ANY . Пример такого элемента:

<!ELEMENT TOVAR ANY>

Использование этого способа не рекомендуется – можно описать любую структуру (в том числе и ошибочную).
