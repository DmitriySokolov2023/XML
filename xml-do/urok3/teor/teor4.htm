4. Ссылка на схему DTD

 

При использовании схемы DTD в заголовке XML документа требуется создать отдельный файл со схемой DTD и в XML документе задать ссылку на этот файл (внешнее расположение) или разместить содержимое схемы в документе XML (внутреннее расположение). Первый способ (внешнее расположение) является более предпочтительным – происходит разделение документа и описания его структуры в разных файлах.

Для задания ссылки на DTD схему, в заголовке XML документа после объявления типа документа (строка начинается со слов <? xml – обычно это первая строка XML -документа) и до объявления корневого элемента указывается следующее предложение –

<!DOCTYPE корневой_элемент [ ... ]>

корневой_элемент – это имя корневого элемента данного документа.

В квадратных скобках указывается содержимое схемы (при внутреннем расположении).

При использовании внутреннего расположения DTD-схемы синтаксис следующий:

<?xml version="1.0" encoding="windows-1251" ?>
<!DOCTYPE TOVARS [
<!ELEMENT TOVARS (TOVAR)>
<!ELEMENT TOVAR (NAIM,PRICE)>
<!ELEMENT NAIM (#PCDATA)>
<!ELEMENT PRICE (#PCDATA)>
]>
<TOVARS>
<TOVAR>
<NAIM>Бензин</NAIM>
<PRICE>20</PRICE>
</TOVAR>
</TOVARS>

В случае внешнего расположения указывается имя корневого элемента, слово SYSTEM и имя файла в двойных кавычках.

Пример описания внешнего расположения схемы:

<!DOCTYPE TOVARS SYSTEM "tovar.dtd">

Описывается ссылка на схему DTD, расположенную в файле tovar.dtd.

При задании ссылки на схему в случае размещения ее в сети Интернет, можно указать URL файла следующим образом:

<!DOCTYPE TOVARS SYSTEM "http://www.site.ru/tovar.dtd">

Как видно, использование внешнего расположения схемы позволяет получить более компактный XML документ.

Указание имени корневого элемента (TOVARS) при задании схемы (как при внутреннем, так и при внешнем расположении) является обязательным условием.
