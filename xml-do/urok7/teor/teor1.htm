Урок 7. Преобразование XML документа с использованием xpath
 

1. Необходимость в преобразовании XML документа

 

Основное назначение XML-документов – это перенос информации между системами обработки данных. Отправляющая система должна обеспечить формирование XML -документа, а принимающая система – его прием и обработку. Часто возникает ситуация, при которой принимающая система должна получить документ определенной структуры, а передающая система формирует документ частично не соответствующий требованиям принимающей системы, причем нет возможности повлиять на исправление процесс формирования файла. Например, требуется получить XML -документ с использованием атрибутов, а передающая система предоставляет документ с использованием элементов.

Также иногда возникает такая ситуация – прайс-лист требуется сформировать в виде XML -файла, но просмотр документа осуществляется при помощи броузера и требуется преобразовать XML-документ в вид, похожий на HTML-документ для просмотра посетителем сайта.. Одним способов решения такой задачи является написание программы-обработчика такого документа одном из скриптовых языков (php, asp и т.д.). У такого способа есть очень существенный недостаток – требуется изучать скриптовый язык, кроме того, при смене провайдера не исключена ситуация и смены скриптового языка, используемого на платформе провайдера. Наконец, такой способ не может быть использован при размещении прайс-листа на одной из бесплатных площадок (например narod.ru). Выходом из этой затруднительной ситуации может быть использование преобразование XML -документа с использованием XPATH.

Суть преобразования XML-документа заключается в том, что создается отдельный файл (называемый XSLT), в котором задаются правила преобразования XML-документа с использованием выражений XPATH. В заголовке XML-документа указывается ссылка на файл XSLT. При открытии в броузере (Internet Explorer, Mozilla, Opera) происходит анализ XML-документа и преобразование его согласно содержимому XSLT файла. Можно создать XSLT файл таким образом, что после преобразования XML документ будет открываться в броузере подобно HTML-документу. Схема XSLT- преобразования показана на рисунке

Преобразование XML-документа с использованием XSL-файла также называется XSL преобразованием (XSLT) выполняется при помощи специального XML приложения, которое является XSL процессором. Этот процессор обрабатывает правила, размещенные в файле XSL. На рисунке показан XSL файл, который содержит одно или несколько шаблонных правил. Каждое правило состоит из 2-х частей:

  1. Правило соответствия, определенное как XPATH -выражение в атрибуте элемента <xsl: elenemt>
  2. Данные преобразования, размещенные между начальным и конечным тегом <xsl: element> элемента и определяющие выходную информацию. Данные преобразования обычно содержат XML/HTML элементы и XSLT правила.

XSL процессор осуществляет поиск элементов из входного XML документа на основе XPATH выражений из правил соответствия в XSL файле. В случае нахождения таких элементов происходит формирование выходного документа на основе преобразованных данных из входного документа с учетом данных из XSL-файла.
