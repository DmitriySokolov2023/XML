5. Задание пользовательского типа для простого элемента

 

При создании XML документов часто возникают случаи, когда несколько элементов имеют схожие ограничения (например должны быть больше заданного значения) или требуется задать типы элементов в начале схемы (для наглядности). В этом случае можно создать пользовательский тип, определить в нем нужные ограничения и затем создать элементы с этим типом. Это можно сделать при помощи элемента схемы simpleType.

Для задания ограничений используется элемент схемы restriction. Например, требуется для информационного содержимого элемента PRICE задать ограничение по максимальному значению (не более 1000).

Реализация этого ограничения приведена на примере 3:

Пример 3

<xs:simpleType name="t_price">

<xs:restriction base="xs:decimal">

<xs:maxInclusive value="1000"/>

</xs:restriction>

</xs:simpleType>

Как видно из примера 3, задается тип (simpleType) с именем t_price с ограничением (restriction), которое основано на базовом типе decimal (число с дробной частью). В этом ограничении определяется максимальное значение – 1000. Это осуществляется при помощи элемента maxInclusive. Часто используемые ограничения показаны в таблице:

Ограничение

Описание

fractionDigits

Количество цифр справа от десятичной точки

Length

Количество символов в строковом ( string ) базовом типе

maxInclusive

Максимальное числовое значение содержимого элемента

maxLength

Максимальное количество символов в элементе со строковым содержимым

minInclusive

Минимальное числовое значение содержимого элемента

minLength

Минимальное количество символов в элементе со строковым содержимым
