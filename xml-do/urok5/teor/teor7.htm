Урок 5. Использование схем XML для описания структуры XML документа
 

7. Задание структуры с несколькими элементами

 

Все приведенные выше структуры и примеры относились к документам, состоящими из одного элемента. Такие документы встречаются на практике очень редко. Чаше всего документ имеет сложную структуру, состоящую из заданной последовательности элементов. Для задания сложной структуры используется элемент схемы complexType , а для задания последовательности элементов – элемент схемы sequence Как и все элементы схемы, в этих элементах указываются префиксы пространства имен.

Для примера можно рассмотреть XML-документ с описанием прайс-листа следующей структуры:

<xml version=” 1.0” encoding=”windows- 1251” ?>

<tovars>

<tovar>Керосин</tovar>

<tovar>Помада</tovar>

</tovars>

Этому документу соответствует следующая схема:

<?xml version="1.0" encoding="windows-1251"?>

<xs:schema xmlns:xs="http://www.w3.org/2001/XMLSchema" elementFormDefault="qualified">

<xs:element name="tovars">

<xs:complexType>

<xs:sequence>

<xs:element name="tovar" maxOccurs="unbounded" type=”xs:string”>

</xs:sequence>

</xs:complexType>

</xs:element>

</xs:schema>

В случае «многоуровневой» структуры документа элементы complextype и sequence могут быть вложенными согласно структуре документа. Для примера можно создать схему для XML -документа с прайс-листом следующей структуры:

<?xml version="1.0" encoding="windows-1251"?>

<tovars>

<tovar>

<naim>Керосин</naim>

<price>10.20</price>

</tovar>

<tovar>

<naim>Помада</naim>

<price>100.20</price>

</tovar>

</tovars>

Этому документу соответствует следующая схема:

<?xml version="1.0" encoding="windows-1251"?>

<xs:schema xmlns:xs="http://www.w3.org/2001/XMLSchema" elementFormDefault="qualified">

<xs:element name="tovars">

<xs:complexType>

<xs:sequence>

<xs:element name="tovar" maxOccurs="unbounded">

<xs:complexType>

<xs:sequence>

<xs:element name="naim" type="xs:string"/>

<xs:element name="price" type="xs:decimal"/>

</xs:sequence>

</xs:complexType>

</xs:element>

</xs:sequence>

</xs:complexType>

</xs:element>

</xs:schema>
